1. Topologia de Pastas e Desacoplamento Arquitetural

O ecossistema foi modularizado de forma estrita sob o princípio de loose coupling (acoplamento fraco), isolando responsabilidades em módulos coesos e auto-suficientes:

• main.c -> Ponto de entrada limpo (Bootstrap de inicialização da pilha). • game.c/h -> Core Engine (Laços aninhados orquestradores da regra de negócio tática). • zombie.c/h -> Camada de Dados (Engenharia da Fila Dinâmica Encadeada e gerência do Heap). • player.c/h -> Camada de Domínio (Estados escalares mutáveis da entidade sobrevivente). • interface.c/h-> Viewport Interface (Abstração de diretivas cross-platform e I/O).

Destaque Técnico: O isolamento estrito via arquivos de cabeçalho (.h) mitigou a ocorrência de conflitos de merge no Git, assegurando o desenvolvimento paralelo.

2. O Nó Dinâmico – Dissecação da struct Zombie

Cada zumbi instanciado na memória RAM funciona como um nó isolado e assíncrono. A tipagem em baixo nível se comporta da seguinte forma:

typedef struct Zombie { char nome[30]; // Linha 2: Alocação estática interna de 30 bytes contíguos para strings int hp; // Linha 3: 4 bytes escalares para a vida corrente int hpMax; // Linha 4: 4 bytes para teto de normalização visual int dano; // Linha 5: 4 bytes para carga ofensiva aplicada na casa zero struct Zombie *prox; // Linha 6: Ponteiro auto-referenciado de 8 bytes (Arquitetura x64) } Zombie; // Linha 7: Instanciação do alias estruturado

Mecânica Microscópica: A linha 6 armazena o endereço binário do próximo bloco dinâmico alocado sob demanda no Heap, abolindo a rigidez dos vetores estáticos.

3. Abstração de Fronteiras da Fila Dinâmica (FIFO)

A estrutura Fila encapsula as fronteiras lineares utilizando o paradigma First In, First Out (FIFO).

typedef struct { Zombie *inicio; Zombie *fim; } Fila;

Remoção no Início em $\mathcal{O}(1)$: O consumo do zumbi ativo ocorre instantaneamente movendo a cabeça da fila, sem deslocar índices físicos.

Inserção na Cauda em $\mathcal{O}(1)$: A manutenção do ponteiro fim elimina a varredura linear do Heap a cada spawn.

4. Alocação em Runtime Dinâmico (enqueueZombie)

A inserção de um nó de zumbi realiza uma chamada de sistema microscópica direto na RAM:

void enqueueZombie(Fila *f, char *nome, int hp, int dano) { Zombie *novo = (Zombie*) malloc(sizeof(Zombie)); // L2: Aloca espaço físico no Heap if (novo == NULL) return; // L3: Proteção preventiva contra Out-of-Memory strcpy(novo->nome, nome); // L4: Deep copy literal dos caracteres novo->hp = hp; novo->hpMax = hp; novo->dano = dano; novo->prox = NULL; if (f->inicio == NULL) { f->inicio = novo; f->fim = novo; } else { f->fim->prox = novo; f->fim = novo; } // L8: Vinculação do ponteiro e avanço da cauda }

5. Coleta de Lixo Manual Preventiva (dequeueZombie)

O Perigo do Nó Fantasma (Memory Leak):

Se avançássemos o ponteiro de cabeçalho diretamente sem resguardar o endereço corrente, o bloco de dados do zumbi morto ficaria isolado no Heap, gerando vazamento crônico de RAM.

void dequeueZombie(Fila *f) { if (f->inicio == NULL) return; Zombie *aux = f->inicio; // L3: Retém a referência física na Stack antes do avanço f->inicio = f->inicio->prox; // L4: A cabeça assume o endereço apontado pelo campo prox if (f->inicio == NULL) f->fim = NULL; free(aux); // L6: Liberação física imediata dos bytes devolvidos ao SO }

6. Inovação: Algoritmo de Waves Infinitas com Rotação Cíclica de 3 Estados

Para evitar a estagnação de dificuldade linear e abolir hordas homogêneas previsíveis, o motor lógico foi atualizado com um algoritmo baseado no **resto de divisão por módulo aritmético** (i % 3):

void iniciarWave(Fila *f, int wave) { int qtdZumbis = wave * 2; // Escalabilidade geométrica infinita da horda for (int i = 0; i < qtdZumbis; i++) { if (wave == 1) { enqueueZombie(f, "WALKER", 30, 8); } else if (wave == 2) { if (i % 2 == 0) enqueueZombie(f, "RUNNER", 40, 12); else enqueueZombie(f, "WALKER", 30, 8); } else { // CICLO INFINITO ROTATIVO ASSÍNCRONO if (i % 3 == 0) enqueueZombie(f, "WALKER", 30, 8); // Passo linear padrão (Avanço 1) else if (i % 3 == 1) enqueueZombie(f, "RUNNER", 40, 12); // Passo dinâmico rápido (Avanço 2) else enqueueZombie(f, "TANK", 80, 20); // Nó massivo (Obriga ciclo de recuo) } } }

Mapeamento Tático: Ao entrelaçar nós adjacentes com pesos de HP discrepantes, o motor força uma quebra contínua na árvore de decisões do jogador. À medida que os zumbis são consumidos na cabeça (FIFO), a velocidade e a pressão sobre o pente mudam em runtime.

7. Matriz de Decisão Espacial – Pista Expandida de 0 a 8 Casas

O combate deixa de ser uma mera subtração estática. O motor orquestra um **Vetor Virtual de Distância Linear Relativa**. O nascimento do zumbi ativa o spawn fixo em 4 casas. A partir daí, cada ação dita uma mutação no espaço:

  • Ação Atacar (Opção 1): Dispara munição, reduz HP e o zumbi avança sua taxa de passos corrente (1 ou 2).
  • Ação Recarregar (Opção 2): Sincroniza o pente (6/6). O turno do player é nulo e o inimigo avança livremente encurtando a horda.
  • Ação Fugir / Recuar (Opção 3): Aplica um modificador incremental geométrico de distancia += 2.
